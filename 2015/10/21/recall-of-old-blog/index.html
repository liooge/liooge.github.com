折腾hexo的时候，才突然记起，gae上面也挂着一个blog，虽然注册的域名早就过了期，但是一直受到gae信件，让备份数据，也没当回事。今天发现所有的数据都是不可访问的了。

借助于伟大的web.archive.org找回了一些片段，片段就片段吧。摘录如下, 完整的东西总是躺在那里，片段才让人唏嘘不已。

  1. openwrt samba failed from windows

    今天心血来潮，在Debian上整了一下samba共享，只实只需几个配置即可，就想到了openwrt ,这上面一直没整成功，就决定用cmd方式挂一下，看一下详细的错误信息，结果原来还有个kmod-fs-cifs呵呵。
    but it still have some problem. I can access to it from my linux ,but for windows ,it still won’t work. Sorry about the message。
    可能是因为xp系统吗？这两天定了一个2G的内存，改天安个Win7 试下。

  2. openwrt samba 共享
    今天心血来潮，在Debian上整了一下samba共享，只实只需几个配置即可，就想到了openwrt ,这上面一直没整成功，就决定用cmd方式挂一下，看一下详细的错误信息，结果原来还有个kmod-fs-cifs呵呵。

  3. debian 6.0 换用记

    昨天在公司电脑上，安装了debian 6.0用了两天后感觉太像ubuntu了。。
    我比较喜欢的gdm setup功能没有了，想在debian6.0实现随机登陆界面估计短时间用不了。。
    之前一直使用debian 5.04，前一段不知安装什么时候把debian 的版本管理搞乱了，以致于后来用新立德动不动就提示key不正确了，without authentication之类错误，当时就感觉到这个系统崩溃只是一个时间问题。。
    话说前几日，在自己电脑ubuntu10.10上使用scim在chrome输入文字时，经常性地出现输入不成功的现象，而且半路出家的五笔玩家，还是比较适应像极点之类的五笔输入法，就试着google “linux 极点”，接着发现原来ibus是这样强大的啊。。
    接着就去网上下ibus-table了，自己创建相应的db了，试了万能五笔感觉很好。。第二天也决定公司电脑换用ibus,安装，但是它怎么也安装不上ibus-table-create-db，整啊整啊，最后点了一下apply，弹出的消息框中，removing libgome**我X了，完了。。
    后来乱整啊，无奈，见识了”Yes, do as i say!”这类强悍的确认符。重装了debian6.0
    之前把/opt挂在了独立的分区，把一些解压即可用的软件都放到那下面去了，如工作用的thunderbird和emesene，把备份的.thunderbird拷至新系统，一切安好。。
    当然像中文字体之类就不说了。。

  4. IPv6 on Openwrt，终于成功！

    我有一个openwrt的路由器，一直想配置ipv6，以享受诸如pt,no-nat的便利，可惜历尽坎坷还是没有成功，今天在公司里使用了一下openwrt x86版本的，OK了，回去试试，可惜不能上网了。

  5. Google Chrome更新了，赞一个

    今天看新闻说google stable系列更新到了9.0.597.94，和往常一样，进新德立，upgrade，完成后，重新启动chrome就是新版本了，window下情况不知道，但在linux（Debian）下，有一个特别大的改进，那就是标题栏的字体放大了，看汉字时，清晰多了，舒服多了。
